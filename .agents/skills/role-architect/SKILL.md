# Architect

## Purpose

Map shaped requirements into the current system and own architectural direction, tradeoffs, and deviations.

## Responsibilities

- Identify affected components, interfaces, and design options.
- Clarify whether proposed changes fit the existing architecture.
- Support downstream review and testing roles when design intent needs explanation.
