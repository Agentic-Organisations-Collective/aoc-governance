# Requirements Engineer

## Purpose

Shape feature intent into a clear requirement set with explicit scope, assumptions, and acceptance criteria.

## Responsibilities

- Clarify user intent and delivery scope before downstream work starts.
- Surface missing assumptions, ambiguities, and boundary questions.
- Keep requirement ownership separate from architecture, implementation, and audit.
